Pipeline Architecture

Both loaders produce the same output shape: (B, T, 3, H, W) float32 on CUDA. fast_loader decodes on GPU hardware (NVDEC), caches in CPU RAM, and uses pinned memory for fast transfer. loader decodes on CPU with cv2, then converts numpy → torch → permute CHW → .to(cuda).

End-to-End Decode: MP4 → GPU Tensor

Full end-to-end: tar extraction + video decode + GPU transfer. The loader path includes torch.from_numpy().permute(0,3,1,2).to("cuda") so the comparison is fair — both end with the same (N, 3, H, W) tensor on CUDA. Measured over 10 episodes.
